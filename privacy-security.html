Privacy & security

Designed around local control and classroom data protection.

Snapshot is built with a local-first philosophy so teachers can manage classroom records without depending on a constant cloud connection.

Security features that support teacher confidence.

The privacy model focuses on keeping classroom data local, protected, and easier to control.

Optional App Passcode

Add a passcode barrier before accessing classroom records.

Manual Lock Mode

Lock the application when stepping away from the computer.

Startup Lock Screen

Require access before opening the teacher workspace.

Salted Passcode Hashing

A safer method for storing passcode verification data.

Optional Local JSON Encryption

Protect local data files with encryption when enabled.

Encrypted Export Bundles

Create protected export bundles with separate export passcodes.

Local-first data philosophy

Snapshot is designed so core classroom information remains available locally and can be managed without requiring a cloud service for everyday use.

Substitute-safe handoffs

The Substitute Teacher Bundle supports limited, read-only classroom handoffs so a substitute can access only the information they need.
