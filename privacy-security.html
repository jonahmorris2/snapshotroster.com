Local-first privacy model

Classroom data should stay under teacher control.

Snapshot is designed around local classroom records, optional app locking, protected exports, and local encryption options so teachers can manage sensitive classroom information with greater confidence.

🔐
🖥️

Local-first storage

Core classroom records are designed for local use instead of requiring a constant cloud connection.

🔑

App access protection

Optional passcode protection, startup lock behavior, and manual lock mode help protect the application when the device is shared or unattended.

🧂

Salted passcode hashing

Passcodes are not treated as plain visible settings. Snapshot uses a safer passcode storage approach.

📦

Protected export bundles

Export workflows can be protected with separate export passcodes for safer data movement.
